# Q&A Defense Preparation

Use this when the user asks for defense preparation, Q&A prediction, teacher questions, or when the presentation type is a defense/report with likely questioning.

## Question Categories

Generate likely questions from these angles:

- Method challenge: why this method, why not another method, sample size, process reliability
- Data challenge: source, date, scope, representativeness, missing comparison
- Causality challenge: whether other factors explain the result
- Boundary challenge: where the conclusion applies and where it does not
- Innovation challenge: what is new compared with existing work or common practice
- Feasibility challenge: whether the solution can actually be implemented
- Limitation challenge: what was not solved and what should be improved next

## Answer Style

For each question, provide:
- concise answer points, not a long script
- one honest limitation when appropriate
- one concrete evidence point from the project, data, or class context
- wording that sounds like a student, not a textbook

## Output Format

```markdown
## Possible Question 1
- Type:
- Why the teacher may ask:
- Answer points:
- Evidence to mention:
- Avoid saying:
```

## Common Defense Phrases

- "感谢老师的提问。这个问题主要涉及..."
- "我们目前的数据还不能完全证明这一点，所以在结论里只保留为..."
- "这个地方确实是后续可以改进的方向。"
- "我们选择这个方法，是因为它更适合本课程项目的时间和样本条件。"
